ch3 类和对象

Object Oriented三个特征

  • 封装性-把对象的属性和行为封装起来，不需要让外界知道具体实现的细节。
  • 继承性-通过继承对原有类的功能进行拓展。
  • 多态性-同一个属性和方法在不同的类中具有不同的语义。

类的封装

类的封装是指在定义一个类时，将类中的属性私有化。

具体为用private修饰私有属性，提供public公用方法来访问和设置私有属性。

构造方法

构造方法也可以重载，当设计者为该类定义了构造方法时，系统就不再提供默认的构造方法了。此时最好重载一个无参的构造方法(类似系统默认的空白构造方法)。

this关键字

即this指代当前对象的属性和方法。

static关键字

静态变量

在一个java类中，可以使用关键字static关键字来修饰成员变量，该变量被称作静态变量。静态变量被所有实例共享，可以使用类名.变量名的形式来访问。

也可以使用该类的实例对象所调用。如S1.school = HENU。

静态变量可以在外部修改。我去我记得c++说静态变量只能被静态函数所修改啊！

PS：static关键字只能修饰成员变量不能修饰局部变量。

静态方法

静态方法可以在再不创建对象的情况下被调用。

在类中定义的方法前加上static关键字即可，调用方法同静态变量一样。

PS：静态方法中只能访问静态变量，因为其他成员需要先实例化。

静态代码块

在java类中,一对大括号包围的一段代码称为一个代码块，用static修饰的代码块称为静态代码块。

当类被加载时，静态代码块会执行，由于类只会加载一次，因此静态代码块也只执行一次。所以我们通常用静态代码块来对静态成员变量进行初始化。

成员内部类

在一个类的内部定义类，这样的类称为内部类。内部类所在的类称作外部类。

成员内部类可以访问外部类的所有成员。且内部类可以在外部类中被使用。

通过外部类去访问内部类，则需要通过外部类对象去创建内部类对象。

4.1类的继承

继承的概念

在java中，类的继承是指在一个现有类的基础上去构建一个新的类，构建出来的新类被称作子类，现有类被称作父类。子类会自动拥有父类所有可继承的属性和方法。使用extends关键字。

Java中继承的一些特点

  • 只支持单继承，不允许多继承，也就是说一个类只能有一个直接父类。

  • 可以多层继承，即A->B->C.

重写父类方法

子类中重写的方法需要和父类被重写的方法具有相同的方法名、参数列表、和返回值类型。

同时，子类总被重写的方法其访问权限不能比父类更严格。如父类是public，子类就不能是private的。

super关键字

super关键字专门用于访问父类的成员。

语法格式与this类似

使用super访问父类的构造函数,直接super作为方法名即可。

PS:通过super调用父类构造方法的代码必须位于子类构造方法的第一行，且只能出现一次！！！

在调用子类的构造方法时一定会调用父类的某个构造方法。我们可以用super指定调用父类的哪个构造方法。如果没有指定，则系统自动调用父类无参的构造方法。所以我们在写父类的构造方法时，尽量自定义一个无参的构造方法。

final关键字

字面理解即无法改变、终止的意思。故被final修饰的类、变量、方法有以下特征。

  • final修饰的类不能继承
  • final修饰的方法不能被子类重写
  • final修饰的变量(成员和局部变量)是常量，只能被赋值一次。

5.1 String类和StringBuff类

二者的不同

  • String 类表示的字符串是常量，一旦创建无法修改。而StringBuff类表示字符容器，其内容和长度可以随时修改。

  • String类覆盖了Object类的equals()方法，而StringBuff类则没有。

  • String 类可以用”+”进行连接，而StringBuff类不能。

事件处理机制

在程序中，如果想实现事件的监听机制，首先需要定义一个实现了事件监听器接口的类，例如在Window类型的窗口需要实现WindowsListener。接着通过addWindowListener()方法为事件源注册事件监听器对象，当事件源上发生事件时，便会触发事件监听器对象，由事件监听器调用相应的方法来处理相应的事件。

大体上有三种方法去实现，老师又对其中一种进行了扩充。

缺点：WindowListener接口有七个方法。都需要一一实现，即使那些没用也要去写。

2.事件适配器

为了解决上述多余但又必须的工作，JDK提供了一些适配器类，它们是监听器接口的默认实现类，这些实现类中实现了接口的所有方法，但方法中没有任何代码，需要自己重写所需要的方法即可，程序可以通过继承类适配器来达到实现监听器接口的目的。
